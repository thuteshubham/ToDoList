Events to listen

verifyUser event
This event ("verify-user") has to be listened on the user's end to verify user authentication.
user will only be set as online user after verification of authentication token.
onlineUsersTodo event
This event ("onlineUsersTodo") has to be listened on the user's end to identify all available users that are currently online.
The output will be an object, object has key as userId and value as full name.
createList-res event
This event ("createList-res") has to be listened to get notification regarding List creation by any user to all users.
deleteList-res event
This event ("deleteList-res") has to be listened to get notification regarding list deletion by any user to all users.
updateList-res event
This event ("updateList-res") has to be listened to get notification regarding list updation by any user to all users.
createItem-res event
This event ("createItem-res") has to be listened to get notification regarding Item creation by any user to all users.
deleteItem-res event
This event ("deleteItem-res") has to be listened to get notification regarding Item deletion by any user to all users.
updateItem-res event
This event ("updateItem-res") has to be listened to get notification regarding Item updation by any user to all users.
markDoneItem-res event
This event ("markDoneItem-res") has to be listened to get notification to all users, if any user mark item as DONE.
markOpenItem-res event
This event ("markOpenItem-res") has to be listened to get notification to all users, if any user mark item as OPEN.
undoItem-res event
This event ("undoItem-res") has to be listened to get notification to all users, if any undo action performed on item.
createSubItem-res event
This event ("createSubItem-res") has to be listened to get notification regarding Sub-Item creation by any user to all users.
deleteSubItem-res event
This event ("deleteSubItem-res") has to be listened to get notification regarding Sub-Item deletion by any user to all users.
updateSubItem-res event
This event ("updateSubItem-res") has to be listened to get notification regarding Sub-Item updation by any user to all users.
markDoneSubItem-res event
This event ("markDoneSubItem-res") has to be listened to get notification to all users, if any user mark sub-item as DONE.
markOpenSubItem-res event
This event ("markOpenSubItem-res") has to be listened to get notification to all users, if any user mark sub-item as OPEN.
undoSubItem-res event
This event ("undoSubItem-res") has to be listened to get notification to all users, if any undo action performed on sub-item.
clearUndos-res event
This event ("clearUndos-res") has to be listened to get notification to all users regarding undos clear OR delete.
reqAlreadySent event
This event ("reqAlreadySent") has to be listened to get notification himself/herself, during second attempt of request, if user has already sent friend request to perticular user.
alreadyFriend event
This event ("alreadyFriend") has to be listened to get notification himself/herself, during second attempt of request, if user has already friend with that perticular user.
savereq event
This event ("savereq") has to be listened to get notification to users, if friend request sent successfully.
rejecting event
This event ("rejecting") has to be listened to get notification himself/herself, after rejecting the friend request of user.
accepting event
This event ("rejecting") has to be listened to get notification himself/herself, after accepting the friend request of user.
removed event
This event ("removed") has to be listened to get notification himself/herself, after removing/unfriend the friend from friend list.
userId event
This event's name is actually dynamic and refers to the userId of the logged in user. This event ("userId") has to be listened to get the updates from Admin user.
Example output it as follows: 
                                    {
                                        userId: 'unique id of User',
                                        data: 'data from user',
                                    }

Events to emit

set-user event
This event ("set-user") has to be emitted when a user comes online.
User can only be set as online only after verification of authentication token. Which you have pass here. The following data has to be emitted:
authentication token
createList event
This event ("createList") has to be emit after creating New List
deleteList event
This event ("deleteList") has to be emit after deleting List.
updateList event
This event ("updateList") has to be emit after updating List.
createItem event
This event ("createItem") has to be emit after creating New Item
deleteItem event
This event ("deleteItem") has to be emit after deleting Item.
updateItem event
This event ("updateItem") has to be emit after updating Item.
markDoneItem event
This event ("markDoneItem") has to be emit after doing Item "Mark As Done".
markOpenItem event
This event ("markOpenItem") has to be emit after doing Item "Mark As Open".
createSubItem event
This event ("createSubItem") has to be emit after creating New Sub-Item
deleteSubItem event
This event ("deleteSubItem") has to be emit after deleting Sub-Item.
updateSubItem event
This event ("updateSubItem") has to be emit after updating Sub-Item.
markDoneSubItem event
This event ("markDoneSubItem") has to be emit after doing Sub-Item "Mark As Done".
markOpenSubItem event
This event ("markOpenSubItem") has to be emit after doing Sub-Item "Mark As Open".
request event
This event ("request") has to be emit after sending friend request to perticular user.
reject event
This event ("reject") has to be emit after rejecting friend request by perticular user.
accept event
This event ("accept") has to be emit after accepting friend request of perticular user.
unfriend event
This event ("unfriend") has to be emit after getting unfriend(removed from friend list) by perticular user.
